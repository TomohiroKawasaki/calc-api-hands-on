# 機能要件（Functional Requirements）

## 1. 目的
Azure Functions（Python 3.11, Linux / Consumption）で、ブラウザから直接アクセス可能な2つのHTTP API（掛け算・割り算）を提供する。

## 2. エンドポイント
- ベースプレフィックス: `api`（既定値）
- API 一覧:
  - `GET /api/multiply` … 掛け算（A × B）
  - `GET /api/divide` … 割り算（A ÷ B）
- 認証レベル: ANONYMOUS（公開）

## 3. 入力仕様
- クエリストリングで受け取る。
  - パラメータ名: `A`, `B`
  - 型: 整数または浮動小数を許容（例: `1`, `-2.5`, `0.1`）
  - 大小文字: `A`, `B` を正とする
  - 言語/ロケール: 小数点は`.`（ドット）

## 4. 出力仕様
- コンテンツタイプ: `text/plain; charset=utf-8`
- ステータスコード: 常に `200 OK`
- 本文:
  - 正常系: 計算結果のみ（例: `6`、`0.5`）
  - エラー系: プレーンテキストでエラーメッセージ（日本語）

## 5. バリデーションと振る舞い
- 必須: `A`, `B` の双方が指定され、数値に変換可能であること
- 変換不可の場合（欠落・非数値など）
  - ステータス: 200
  - 本文例: `入力エラー: A または B が数値ではありません`
- `divide` で `B=0` の場合
  - ステータス: 200
  - 本文: `0除算です`
- 返却値のフォーマット
  - 計算は Python の数値演算に準拠
  - 末尾の不要な小数部の扱いは実装既定（例: `6.0` をそのまま出力）

## 6. 例
- 掛け算（正常）
  - リクエスト: `GET /api/multiply?A=2&B=3`
  - レスポンス: `200` / 本文: `6`
- 割り算（正常）
  - リクエスト: `GET /api/divide?A=1&B=2`
  - レスポンス: `200` / 本文: `0.5`
- 入力エラー（数値でない）
  - リクエスト: `GET /api/multiply?A=foo&B=3`
  - レスポンス: `200` / 本文: `入力エラー: A または B が数値ではありません`
- 0除算
  - リクエスト: `GET /api/divide?A=1&B=0`
  - レスポンス: `200` / 本文: `0除算です`

## 7. 非対象（スコープ外）
- JSON 応答、HTML テンプレートなどのリッチ表示
- POST/PUT 等の他 HTTP メソッド対応
- 小数点以外のロケール（カンマ区切り等）
- 四捨五入や桁数固定などの数値整形ルール
